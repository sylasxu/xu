# Skill: Web Chat UI（社交 Agent 输入体验）

## 适用场景

当任务涉及以下任一情况时使用本 skill：

- 修改 `apps/web/app/chat/**` 或 `apps/web/components/ai-elements/**`
- 调整输入框、建议问题区、欢迎区/卡片区视觉
- 优化移动端 H5 聊天交互

## 目标

围绕“高频输入效率 + 清晰可读 + 轻量视觉”优化社交 agent 聊天页。

## 执行清单

1. 输入框状态机（必须）
   - 默认：单行布局（紧凑、降低视觉噪音）
   - `focus`：扩展为双行布局（上层辅助操作 + 下层主输入）
   - 发送中：保留输入上下文，避免布局抖动
2. 对齐与层级（必须）
   - 问候文案与功能 icon 顶部对齐（左上/右上）
   - 建议问题卡片优先可点面积与可读性，不堆文案标签
3. “我的数据”卡片（可选但推荐）
   - 优先展示有接口来源的数据（参与次数、发起次数、偏好完善度等）
   - 不展示空口号型标签（如无信息增量的“今日推荐”标题）
4. 视觉风格（推荐）
   - 采用轻玻璃感：半透明背景 + 细边框 + 轻内阴影
   - 避免重阴影与高饱和装饰，保证信息优先级
5. 可访问性与移动端约束（必须）
   - 输入字号 ≥ 16px（移动端）
   - 焦点态可见（`focus-visible`）
   - 点击热区合理（移动端最小 44px）

## 不要做

- 本任务未要求时，不引入夜间模式链路
- 不改动 `/ai/chat`、`/ai/welcome` 协议语义
- 不引入页面私有协议与后端分叉字段

## 仓库内快速检查命令

```bash
rg -n "focus-visible|text-\\[16px\\]|/ai/chat|/ai/welcome|glass|backdrop" apps/web
```

## 产出标准

- 必须描述 idle/focus/sending 三态行为
- 必须说明“展示的数据字段来自哪里”
- 必须给出至少 1 张页面关键区块的改造说明
